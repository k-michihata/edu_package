開発フェーズ移行時に改めて選定・確定する。

## フロントエンド候補

| 技術 | 特徴 | 向いているケース |
|------|------|----------------|
| React | コンポーネント設計・エコシステムが豊富 | Webアプリ全般 |
| Next.js | React基盤・SSR対応・Vercelへのデプロイが容易 | Webアプリ、SEO不要な管理画面含む |
| Vue.js | 学習コストが低め・日本語ドキュメントが充実 | 小〜中規模のWebアプリ |

### 人生グラフの描画
| 技術 | 特徴 |
|------|------|
| Recharts | React向け・シンプルな折れ線グラフに向いている |
| Chart.js | 汎用性が高く、軽量 |
| D3.js | 高度なカスタマイズが可能・学習コストは高め |

---

## バックエンド候補

| 技術 | 特徴 | 向いているケース |
|------|------|----------------|
| Node.js (Express / Hono) | JavaScript統一・軽量 | フロントエンドと言語を統一したい場合 |
| Python (FastAPI) | AI連携がしやすい・シンプルな記述 | AI機能を多用する場合 |
| Firebase | サーバーレス・認証・DBが一体 | 小規模・素早くプロトタイプを作りたい場合 |
| Supabase | Firebase代替・PostgreSQL・認証機能あり | オープンソースを好む場合 |

---

## データベース候補

| 技術 | 特徴 |
|------|------|
| PostgreSQL | リレーショナル・安定性が高い |
| MySQL | 広く使われているリレーショナルDB |
| Firestore | NoSQL・リアルタイム同期・Firebaseと統合 |
| Supabase (PostgreSQL) | PostgreSQLベース・管理UIが使いやすい |

---

## 認証

| 技術 | 特徴 |
|------|------|
| Google OAuth 2.0 | Google Workspace for Educationとの連携に最適 |
| NextAuth.js | Next.js向けの認証ライブラリ・Google OAuth対応 |
| Firebase Authentication | Google認証を含む複数プロバイダに対応 |
| Supabase Auth | Google OAuth対応・Supabaseと統合 |

### 方針
- 学校アカウント（Google Workspace for Education）でのSSOを実現するため、**Google OAuth 2.0ベース**の認証が必須
- バックエンド選定に合わせて認証ライブラリを選ぶ

## AI連携

| 項目 | 候補 | 備考 |
|------|------|------|
| AIモデル | OpenAI API（GPT-4o等） / Anthropic API（Claude等） | 事象生成・行動タイプ別フィルタリングに使用 |
| 利用方法 | APIコール（バックエンド経由） | フロントエンドから直接呼び出しはセキュリティリスクあり |
| プロンプト管理 | バックエンド側で管理 | 行動タイプ・生徒入力事象をプロンプトに組み込む |

### AI事象生成のプロンプト設計イメージ
```
- 入力：行動タイプ（dominant_type）、生徒入力事象（UserEvent）、対象年代（前半/後半）
- 出力：事象リスト（5個）
  - 生徒入力から最低1つを含む
  - 行動タイプに応じた種類・優先順位で補完
  - 社会変化・ライフイベントを混在
  - ポジティブ・ネガティブを混在
  - 「●歳のとき、○○が起こる」形式で出力
```

## 技術選定における考慮事項

| 項目 | 考慮内容 |
|------|---------|
| 学校環境 | PC・タブレット・ChromebookなどデバイスはWebブラウザで動作することが望ましい |
| 開発コスト | 個人開発の場合はFirebase / Supabaseなどのサーバーレスが現実的 |
| AI利用コスト | APIコールのコストを考慮し、生成タイミング・回数を最適化する |
| セキュリティ | 生徒の個人情報・回答データの保護（APIキーのサーバーサイド管理など） |
| 拡張性 | 将来的なパッケージ追加（案B・案C）を見据えた設計 |
