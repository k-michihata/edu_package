# 04 機能要件

## 機能一覧

### 認証・アカウント管理

| ID | 機能 | 説明 | 優先度 |
|----|------|------|--------|
| F-01 | 学校アカウントログイン | Google Workspace for Education等のSSOでログイン | 必須 |
| F-02 | セッション継続 | ログインすることで前回の操作状態を引き継ぎ再開できる | 必須 |
| F-03 | ロール管理 | 生徒・教師（管理者）のロールを区別する | 必須 |

---

### 自己分析フェーズ

| ID | 機能 | 説明 | 優先度 |
|----|------|------|--------|
| F-10 | 価値観記述 | 教師がカスタマイズした問いかけに対してテキストで自由記述する | 必須 |
| F-11 | キャリアアンカー診断 | 高校生向けに読み替えた8項目の診断を実施し行動タイプを確定する | 必須 |
| F-12 | 行動タイプ保存 | 診断結果（行動タイプ）をアカウントに紐づけて保存する | 必須 |

---

### シミュレーションフェーズ

| ID | 機能 | 説明 | 優先度 |
|----|------|------|--------|
| F-20 | 将来事象の入力 | 「●年後に○○が起こる」形式でカテゴリ別に3〜5個入力する | 必須 |
| F-21 | AI事象生成 | 行動タイプと生徒入力事象をもとにAIが事象を生成する（合計5個） | 必須 |
| F-22 | 生徒入力事象の反映 | AIが生成する5個の中に生徒入力事象を最低1つ含める | 必須 |
| F-23 | 行動タイプ別優先順位調整 | 行動タイプに応じてAIが提示する事象の種類・優先順位を変える | 必須 |
| F-24 | ポジ/ネガ評価 | 各事象に対してポジティブ/ネガティブの評価を回答する | 必須 |
| F-25 | 行動記述 | 各事象に対して自分の行動をテキストで記述する | 必須 |
| F-26 | 前半・後半の分割 | シミュレーションを10〜40代（前半）と50〜80代（後半）に分けて実施する | 必須 |
| F-27 | シミュレーションデータ保存 | 各事象への評価・記述をアカウントに紐づけて保存する | 必須 |

---

### 人生グラフ

| ID | 機能 | 説明 | 優先度 |
|----|------|------|--------|
| F-30 | 人生グラフ生成 | シミュレーション結果をもとに人生グラフを生成・表示する | 必須 |
| F-31 | 横軸：年齢/ライフステージ | 横軸に年齢・ライフステージを表示する | 必須 |
| F-32 | 縦軸：感情のポジ/ネガ | 縦軸に感情のポジ（＋）/ネガ（ー）を表示する | 必須 |
| F-33 | 価値観領域マップ | 仕事・家族・健康などの領域別にポジ/ネガを可視化する（拡張） | 拡張 |

---

### アドミン機能（教師）

| ID | 機能 | 説明 | 優先度 |
|----|------|------|--------|
| F-40 | 進捗確認 | クラス・生徒ごとの進捗状況を一覧で確認する | 必須 |
| F-41 | 問いかけカスタマイズ | 価値観記述フェーズの問いかけ内容を編集・保存する | 必須 |
| F-42 | 人生グラフ一覧 | 生徒全員の人生グラフを一覧で確認する | 必須 |

---

## 非機能要件

| 項目 | 要件 |
|------|------|
| 対応言語 | 日本語 |
| 対象地域 | 日本 |
| 対象デバイス | 未定（PC・タブレット・スマートフォンを想定） |
| データ保持 | ログイン単位でデータを永続保存する |
| セキュリティ | 生徒の個人情報・回答内容を適切に保護する |
| アクセシビリティ | 高校生が直感的に操作できるUI |

---

## 将来的な拡張要件

| ID | 機能 | 説明 |
|----|------|------|
| E-01 | シミュレーション先行型（案B） | 事象提示から始まり回答の積み重ねで価値観を導くパッケージ |
| E-02 | 並行型（案C） | 自己分析とシミュレーションを往復しながら深めるパッケージ |
| E-03 | 価値観領域マップ | 人生グラフに加えて領域別の可視化を追加する |
